# 커밋 루프 (전역 작업 규칙)

코드 변경 작업은 아래 순서로 진행한다. 단순 질문·조사·설명, 그리고 아주 사소한 한두 줄 수정(오타·상수·로그 등)에는 적용하지 않고 바로 처리한다.

1. 큰 작업은 먼저 순서 있는 "커밋 단위" 목록으로 쪼개 제시하고 **멈춰 승인을 기다린다.** 각 커밋은 독립적으로 리뷰/되돌리기 가능한 단위로 만든다. 분해안엔 **셀프 리뷰**(리스크·버린 대안·누락)를 함께 붙이고, **큰/아키텍처성 작업**이면 구현 전 접근을 한 번 더 검증한다(필요하면 별도 리뷰).
2. 각 커밋 단위마다:
   a. 이번 커밋의 변경 계획을 제시하고 **멈춰 승인을 기다린다.**
   b. 승인 범위만 구현한다.
   c. repo의 빌드/테스트를 실행한다(명령은 repo의 `.githooks/pre-commit` 또는 repo AGENTS.md를 따름). 실패하면 통과까지 스스로 고친다.
   d. `/review`로 변경분 리뷰를 받아 결과를 보여준다.
   e. **멈춰서 내 확인을 기다린다.** 수정 요청이면 반영, 아니면 다음으로.
3. **확인 게이트(2e)를 통과하기 전에는 스스로 커밋하지 않는다.** 내 확인을 받은 뒤엔 커밋해도 된다. 단, 절대 스스로 다음 커밋 단위로 넘어가지 않는다.
