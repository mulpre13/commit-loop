## 커밋 루프 (코드 변경 기본 규칙)

사소하지 않은 코드 변경은 아래 루프로 진행한다. **예외**(루프 없이 바로 처리): 단순 질문·조사·설명, 그리고 아주 사소한 한두 줄 수정(오타·상수·로그 등).

1. 큰 작업은 순서 있는 "커밋 단위"(독립적으로 리뷰/되돌리기 가능)로 쪼개 제시하고 **멈춰 내 승인을 기다린다.** 분해안엔 **셀프 리뷰**(리스크·버린 대안·누락)를 함께 붙이고, **큰/아키텍처성 작업**이면 구현 전 접근을 한 번 더 검증한다(필요하면 별도 리뷰).
2. 각 커밋 단위마다: ① 변경 계획 제시 후 **멈춰 승인** → ② 승인 범위만 구현 → ③ repo 빌드/테스트 실행(`.githooks/pre-commit` 또는 repo CLAUDE.md를 단일 소스로; 실패 시 통과까지 스스로 수정) → ④ `code-reviewer` 서브에이전트로 리뷰 → ⑤ **확인 게이트: 멈춰서 내 OK를 기다린다.**
3. **내 승인(확인) 없이는 스스로 커밋하지 않는다.** 확인을 받은 뒤엔 커밋해도 되지만, 절대 스스로 다음 커밋 단위로 넘어가지 않는다.
